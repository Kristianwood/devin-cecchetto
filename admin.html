Devin’s admin panel

Edits publish straight to the live site (it takes about a minute to go live). To unlock the panel you need a GitHub access token — a one-time setup. Which kind depends on the account you use:

  1. If your GitHub account owns the site (Kristianwood): open github.com/settings/personal-access-tokens/new · Repository access: Only select repositories → devin-cecchetto · Permissions → Repository permissions → Contents: Read and write (the dropdown defaults to Read-only — change it!) · Expiration: 1 year
  2. If you were added as a collaborator (e.g. Devin on her own account): fine-grained tokens won’t work for a site owned by someone else — create a classic token instead at github.com/settings/tokens/new · tick the repo scope · Expiration: 1 year. (The site owner must first add your account under the repository’s Settings → Collaborators.)
  3. Generate, copy the token and paste it below (it stays in this browser only)
